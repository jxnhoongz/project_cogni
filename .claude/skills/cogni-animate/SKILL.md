# Cogni â€” animate scenes with Higgsfield (camera move on the still)

Each scene flagged **animate=true** becomes a moving hero clip generated by Higgsfield
**Seedance 2.0** from its **single start still** (`scene_XXX.png`) plus the scene's
`video_prompt` â€” a real, slow **camera move** (push-in, pan, parallax, â€¦). The clip
drops into `projects/<book>/clips/scene_XXX.mp4`, and `assemble` uses it (played once,
then holds the last frame) instead of the still. Un-flagged scenes stay Ken Burns stills.

Why single-still, not two keyframes: near-identical start/end frames froze the clips â€”
see `docs/motion.md`. One still + a camera-move prompt gives real motion and costs half
the images. (`end_image_prompt`/end stills may still exist in the schema; ignore them.)

The project leans toward animating **every** scene, but cost scales with total animated
seconds â€” so **start small when testing** (one scene), confirm it moves, then scale up.

## Steps

1. **See what's flagged and ready:**
   ```
   .venv\Scripts\python.exe main.py animate
   ```
   Lists each animate scene, its start still, `video_prompt`, and target clip path.

2. **Preflight the exact cost, then quote it â€” non-negotiable.** Cost â‰ˆ **3.5 credits/sec**
   (Seedance 2.0, 720p, fast). Call `generate_video` with `get_cost: true` and the same
   params as below to confirm, multiply by scenes Ã— seconds, show `balance`, and **wait
   for the user's explicit "go".** First test = one scene.

3. **For each flagged scene** â€” one still in, camera move out:
   1. Upload the start still (`images/scene_XXX.png`): `media_upload` â†’ `curl -X PUT`
      the bytes â†’ `media_confirm` â†’ `start_media_id`.
   2. `generate_video`:
      - `model`: `seedance_2_0`
      - `medias`: `[{ "value": <start_media_id>, "role": "start_image" }]`  (single image!)
      - `prompt`: the scene's **`video_prompt`** verbatim â€” a real slow camera move. Do
        NOT add "no camera shake", "preserve the still", "static", etc. (they freeze it).
      - `duration`: **6** (minimum 5 â€” Seedance ignores motion below 5s; up to 15 to cover
        more of the narration).
      - `params`: `resolution: "720p"`, `mode: "fast"`, `aspect_ratio: "16:9"`,
        **`generate_audio: false`** (we add our own narration).
   3. Poll `job_status({ jobId, sync: true })` until done; take **`rawUrl`**.
   4. Save it:
      ```
      .venv\Scripts\python.exe -c "from cogni.animate import save_clip; save_clip(<scene_id>, '<rawUrl>')"
      ```

4. **Re-assemble** once the clip(s) are saved:
   ```
   .venv\Scripts\python.exe main.py assemble --force
   ```
   Animated scenes play their clip (then hold the last frame for the rest of the scene);
   the rest stay Ken Burns stills.

## Guardrails

- **Start small / quote first.** Always `get_cost` and get an explicit "go" before
  spending. When testing, animate ONE scene and review it before scaling.
- **Single start image + a real camera move.** No end frame. If the `video_prompt` reads
  static ("gentle drift", "no shake"), it's an old prompt â€” re-run `visuals` first.
- **Clips â‰¥ 5s** or Seedance ignores the motion.
- **Silent clips.** Always `generate_audio: false` â€” the video carries our own TTS.
- If a generation errors mysteriously, suspect **plan-tier gating** or a lapsed
  subscription (check `/mcp` and `balance`) before debugging the prompt.
